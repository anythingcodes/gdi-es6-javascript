History

ES6 = ECMAScript 2015

ES7 = ECMAScript 2016

ES.Next = Any upcoming version of ECMAScript


  • ES6 is sometimes called ECMAScript 2015 because it was released in 2015. ES6 and ECMAScript are the same thing. This is true for ES7, a.k.a. ECMAScript 2016.
  • ES6 gets the most attention because it was a huge spec overhaul. It introduced a ton of new features. ES7 and all versions going forward will be smaller, more incremental changes; ES7 only introduced two new features, so we'll discuss them as part of this class.

The Pros

Tons of new features that will help us:

 

  • Cut down on unnecessary code
  • Load pages and apps faster
  • Make our code much cleaner and easy to read, which is perfect for developing in a team setting
  • More easily follow best coding practices
  • Stay ahead of the curve, professionally
Ultimately, makes us better, more professional developers. If you bust out ES6/7 in an interview, they're going to know you're staying ahead of the curve.
